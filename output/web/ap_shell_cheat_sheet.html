Section B.1 Bash Shell Cheat Sheet

Getting help with shell commands.

man <command-name>

Use built-in manual.

  • e.g. Retrieve manual for changing directories command: man cd

<command-name> --help

Request help page for specified command.

  • e.g. Retrieve help for change directory command: cd --help

File and Directory Commands.

pwd

Print working directory displays the name of the current working directory.

  • e.g. Print the path of the current working directory: pwd

whoami

The whoami prints the userid of the current user.

  • e.g. Print the userid: whoami

ls

List displays basic information about files and directories.

  • e.g. List directories and files in the current directory: ls

  • e.g. list directories and files in the current directory using a long listing: ls -l

  • e.g. list directories and files in home directory: ls ~

touch <file-name>

The touch is commonly used for file creation. Its intended primary function is to update its timestamp, by "touching" it. See man touch for more information on the intended use.

  • e.g. touch newfile.txt

cd <directory-name> or relative position

Change directory to <directory-name>.

  • e.g. change directory to the root directory: cd /

  • e.g. change directory to the home directory: cd ~

  • e.g. change directory to the immediate parent directory: cd ..

mv <old-name> <new-name>

Move (rename) files or directories.

  • e.g. Change the name of old.txt to new.txt: mv old.txt new.txt

rm <file-name>

Remove deletes a file or directory.

  • e.g. Remove the file named junk.txt: rm junk.txt

mkdir <directory-name>

Make directory with parameter name <directory-name>.

  • e.g. Make a new directory with the name newdir: mkdir newdir

rmdir <directory-name>

Remove directory with specified name <directory-name>

  • e.g. Remove (delete) the directory with the name olddir: mkdir olddir

The Basics: Reading, Writing, Counting, etc.

echo <text>

The echo command displays a line of text and/or the value of a variable. Often used with output redirection.

  • e.g. Print the text 'Hello World!' on the screen: echo 'Hello World!'

  • e.g. Print the value of the USER environment variable on the screen: echo $USER

cat <file-name>

The concatenate prints file contents on the screen after concatenation. Note that with a single file, it just prints that file. It is often used with output redirection.

  • e.g. Print the contents of file.txt on the screen: cat file.txt

  • e.g. Print the contents of the concatenation of file1.txt and file2.txt on the screen: cat file1.txt file2.txt

read <variable-name>

The read command reads a line or variable from the keyboard. It is often used with scripts or input redirection.

  • e.g. Take input from the keyboard and direct it into a variable called MYVAR: read MYVAR

wc <file-name>

The word count command performs a count of lines, words, and bytes for each file.

  • e.g. Report the count of lines, words, and bytes in file.txt: wc file.txt

Input and Output Redirection.

Input redirection using <

Input redirection uses using <to allow the user to redirect the input from a file rather than the keyboard.

  • e.g. Perform the wc command on the information in the file info2count.txt: wc < info2count.txt

Output redirection using > or >>

Output redirection allows the user to redirect the output from the screen to a file using > for overwriting or >> for appending.

  • e.g. Write the line 'I love open source!' into the file file.txt replacing the current contents or making a new file if it doesn't already exist: echo 'I love open source!' > file.txt

Piping |

A pipe | in the bash shell allows you to redirect (pipe) the output of one command into the input of another command.

  • e.g. Run the command ls>and use the output of the ls command as the input into the wc command: ls | wc

File Permissions.

chown <newuser:newgroup> <file-name>

The chown command is used to change the file owner and/or group.

  • e.g. Change the owner of file.txt to pearcej: chmod pearcej file.txt

  • e.g. Change the group of file.txt to friends: chmod :friends file.txt

chmod <change><which> <file-name>

The chmod command is used to change permissions. The following symbols are the most commonly used:

+ change by adding permission

- change by removing permission

r which permission: read

w which permission: write

x which permission: execute

  • e.g. The following adds execute permission for all users to the helloworld.sh file: chmod +x helloworld.sh
